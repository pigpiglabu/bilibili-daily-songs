# B站 AI 每日歌单推荐插件

一个 Chrome Manifest V3 插件：
- 用户在弹窗里输入音乐偏好与今日心情；
- 插件自动扩展关键词（模拟 AI 提示词增强）；
- 在 B 站页面拉取音乐视频搜索结果；
- 每天按固定种子生成「每日 10 首」推荐，保证同一天结果稳定。

## 功能说明

1. **偏好配置**
   - 在扩展弹窗设置关键词（例如：华语流行, 女声, 治愈）
   - 选择心情（放松/专注/活力/怀旧）

2. **AI 组歌单逻辑（本地）**
   - 根据心情扩展语义关键词
   - 生成推荐提示词（记录在控制台）
   - 聚合多个关键词的 B 站视频搜索结果
   - 去重后按日期种子随机，截取 10 条

3. **每日推荐展示**
   - 访问 `https://www.bilibili.com/*` 页面时自动展示浮层
   - 展示 10 首歌曲视频，点击直达视频页

## 怎么用（不用打包也能用）

> 结论：**自己本地使用不需要打包**，直接“加载已解压的扩展程序”即可。

1. 打开 Chrome 扩展管理页：`chrome://extensions/`
2. 开启右上角「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择本仓库目录（包含 `manifest.json` 的目录）
5. 点击浏览器右上角插件图标，打开 **B站 AI 每日歌单**
6. 输入关键词并选择心情，点“保存偏好”
7. 打开/刷新任意 B 站页面，右下角会出现当日推荐浮层

## 需要打包吗？

### 场景 A：你自己用
不需要打包，按上面的“加载已解压”方式最方便。

### 场景 B：发给别人离线安装
可以打成 zip 给别人，别人解压后仍用“加载已解压”安装。

示例命令（在仓库根目录执行）：

```bash
zip -r bilibili-daily-songs-extension.zip manifest.json src README.md
```

### 场景 C：上架 Chrome Web Store
需要按商店要求准备发布包（通常是 zip），并在开发者后台提交审核。

## 常见问题

- **保存偏好后没显示推荐？**
  - 确认当前页面是 `https://www.bilibili.com/*`
  - 刷新页面后再看右下角浮层
  - 在扩展弹窗重新保存一次偏好

- **推荐结果不够“AI”？**
  - 当前是“轻量 AI 逻辑”（关键词扩展 + 稳定随机）
  - 可后续接入真实 LLM API 提升语义理解与排序质量

## 目录结构

- `manifest.json`：扩展声明
- `src/popup.html`：偏好设置页面
- `src/popup.js`：偏好存储与关键词扩展
- `src/content.js`：B站页面内的每日推荐逻辑与浮层渲染

## 备注

- 当前实现为“轻量 AI 逻辑”（规则+关键词扩展+每日种子随机）。
- 如果需要接入真实 LLM，可在 `popup.js` 或 `content.js` 中增加 API 调用（需配置密钥与接口）。
